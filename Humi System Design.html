01 / Foundations

ระบบ HR สำหรับพนักงานหน้าร้านและผู้จัดการสาขา

Humi ลดงานเอกสารของพนักงานและผู้จัดการสาขาให้กลายเป็นการกดเพียงไม่กี่ปุ่ม โดยรวมการลา เงินเดือน เป้าหมาย การเรียนรู้ และประกาศของบริษัทไว้ในจุดเดียว

Active employees1,840ทั่วทั้งภาคตะวันออก
Stores onboarded42เป้าหมายปี 2569 → 60
Daily active rate76%เข้าใช้อย่างน้อย 1 ครั้ง/วัน

หลักการออกแบบ

01

เร็วกว่ากระดาษ

ทุก task ที่พนักงานเคยทำผ่านกระดาษต้องเสร็จได้ใน <3 ครั้งกด ดีไซน์ขับเคลื่อนด้วย default ที่ฉลาดและการกรอกล่วงหน้า

02

อบอุ่น ไม่เหมือนระบบราชการ

ใช้สีครีม/ฟอนต์มีบุคลิก รูปทรง blob และ tag ที่นุ่มนวล เพื่อให้รู้สึกเหมือนแอปสำหรับมนุษย์ ไม่ใช่ระบบราชการ

03

ภาษาไทยเป็นพลเมืองชั้นหนึ่ง

Type scale, line-height และ letter-spacing ปรับสำหรับสคริปต์ไทย ใช้ฟอนต์ CPN เป็นหลัก พร้อม fallback IBM Plex Sans Thai

สถาปัตยกรรมระดับสูง

Layer 4 · Screens หน้าจอเฉพาะโดเมน — ลา เงินเดือน เป้าหมาย ประกาศ ฯลฯ screens/*.jsx
Layer 3 · Shell Sidebar + Topbar + nav state · เปลี่ยน screen ผ่าน onNav shell.jsx
Layer 2 · Primitives Button · Card · Tag · Tabs · Toggle · Avatar · Ring · Calendar · Field · List styles.css
Layer 1 · Tokens Color · type · radius · shadow · spacing · grain · blob styles.css :root
Layer 0 · Brand โลโก้ Humi · ShelflyMark · accent hue (Tweakable) icons.jsx

กฎการใช้

  • Display ใช้สำหรับ heading และ KPI numbers เท่านั้น — ไม่ใช้กับ paragraph
  • ตัวเลข KPI ใช้ font-family: var(--font-display); font-weight: 700
  • Eyebrow ทุกตัว uppercase + letter-spacing: 0.08–0.14em · ใช้นำ heading เพื่อสร้าง rhythm
  • ภาษาไทย: ใช้ font-feature 'ss01','cv11' เพื่อปรับสระให้ optical balance

Forms

งาน HR ส่วนใหญ่คือการ "กรอกแล้วส่งให้ใครสักคนอนุมัติ" — บทนี้กำหนดวิธีประกอบฟอร์มให้รู้สึกเหมือน Humi ไม่ใช่ admin tool ของบริษัท

Anatomy

ฟอร์มของ Humi ไม่ใช้ label นอก field แบบสองคอลัมน์แบบเอกสารราชการ ใช้ .field แบบวางบนสุดของ input และจัดกลุ่มด้วย card เดียว · ปุ่ม action อยู่ล่างสุดของการ์ดและมี balance/preview ทางขวา

3
Field คู่ใช้ form-row-2

Date range, time range, currency+unit — จับเป็นแถวเดียวกัน เพราะอ่านเป็นความหมายเดียว · ไม่จับคู่ field ที่ไม่เกี่ยวกัน

4
Optional ระบุชัด · ไม่ใช่ดาว

Required คือ default — ใส่คำว่า "(ไม่บังคับ)" ในวงเล็บที่ label ของ field ที่ optional แทน · ไม่ใช้สีแดงดาว

5
Action bar + approver hint

ปุ่ม primary + ghost (บันทึกร่าง) · ขวาสุดเป็น metadata บอกว่าใครจะรับเรื่อง — ลด anxiety ก่อนกดส่ง

Form rules

Rule · 01Top label, never inlineLabel อยู่บน input · ไม่มี placeholder ทำหน้าที่ label · accessibility + RTL safe
Rule · 02Required is the defaultใช้ (ไม่บังคับ) ที่ field optional แทนการใส่ * ที่ required
Rule · 03Validate on blurไม่ validate ทุก keystroke · error เคลียร์ทันทีที่เริ่มแก้ใหม่
Rule · 04Always autosave draftsทุกฟอร์มมีปุ่ม "บันทึกร่าง" + autosave หลัง 3 วิ idle · ดึงคืนได้ที่ Requests/ของฉัน
Rule · 05Show approver before submitลด anxiety · ระบุชื่อ + role ของผู้อนุมัติถัดไป — ไม่ใช่แค่ "หัวหน้า"
Rule · 06Errors are kind"ระบุวันที่เกิดค่าใช้จ่าย" ดีกว่า "Field is required" · บอกการแก้ ไม่ใช่บอกความผิด

Do / Don't (กันใช้ปะปน)

Do · Toggle

ใช้ Toggle เฉพาะ setting ที่ apply ทันที เช่น dark mode, push notification — ไม่ต้องมีปุ่ม Save

Don't · Toggle

อย่าใช้ Toggle ในฟอร์มที่ผู้ใช้ต้องกด Submit — ใช้ Checkbox แทน เพราะ Toggle สื่อว่ามีผลทันที

Do · Radio

Radio = ตัวเลือกเห็นได้ทั้งหมดในจอ ผู้ใช้ scan แล้วเลือก 1 (≤4 ตัวเลือก)

Don't · Radio

อย่าใช้ Radio เกิน 5 ตัวเลือก — เปลี่ยนเป็น Select dropdown ทันที

Do · Date range

ใช้ปฏิทิน 1 ตัวที่ paint ช่วงให้เห็น + preset shortcut (วันนี้, 7 วัน, เดือนนี้)

Don't · Date range

อย่าใช้ Date picker 2 อันแยกกัน — ผู้ใช้พิมพ์วันสิ้นสุดก่อนวันเริ่มได้ และไม่เห็นช่วง

Do · Multi-select

ใช้ดรอปดาวน์ + chip + search ในตัว สำหรับ list >6 ตัว — ติ๊ก ✓ ในเมนูเพื่อเลือกหลายค่า

Don't · Multi-select

อย่าใช้ Checkbox group เมื่อมี >6 ตัวเลือก — เปลือง real estate และ scan ยาก

Spec ทุก input ใช้ร่วมกัน

Sizing ความสูง 36 / 40 / 48
36px = compact (table inline) · 40px = default · 48px = touch / mobile · padding-x 12px ทุกขนาด
States 5 สถานะมาตรฐาน
default · hover · focus (4px ring var(--accent-soft)) · disabled (opacity .5) · error (border var(--coral) + ring 16%)
Label บนเสมอ
Label อยู่บน field · 13px / weight 600 · helper text 12px ใต้ field · error message แทน helper เมื่อ invalid
Required * แดง · ไม่ใช้ "(optional)"
บังคับ = "*" สีคอรัลหลัง label · ฟิลด์ optional ไม่ต้อง mark · ลด noise
Width ตามเนื้อหา
วันเดือนปี = 180px · time = 120px · email/text = full · จำนวน = 100px · ห้ามยืด field สั้นให้เต็มแถว

pending→manager_reviewผู้จัดการเปิดดู (ครั้งแรก)ตั้ง timer SLA 24 ชม.
manager_review→approvedกด "อนุมัติ"หัก balance · sync POS · push พนักงาน
manager_review→rejectedกด "ปฏิเสธ" + เหตุผลคืน balance · push พนักงาน + เหตุผล
approved→paidระบบส่งไป B-PLUS เดือนถัดไปแสดงบน payslip · ปิด case
pending→escalatedSLA > 48 ชม. ไม่มีการดูส่งให้ผู้จัดการเขต · alert HR
draft→cancelledพนักงานถอนคำขอเองลบ draft · ไม่มี audit

Edge cases

!
ลาในวันที่มีกะแล้ว

ตรวจ overlap กับตาราง POS · ถ้ามีกะ → บังคับเลือกผู้ทดแทนจาก dropdown ก่อนส่งได้

!
โควต้าไม่พอ

คำนวณ days_used + คำขอที่อยู่ใน queue · ถ้าเกิน → block ส่ง · เสนอ "เปลี่ยนเป็นลาไม่รับเงิน"

!
ผู้จัดการลาด้วย

ถ้า manager_id อยู่ใน leave ตอนรับเรื่อง → routing อัตโนมัติไป manager ของ manager (เขต)

!
ขอย้อนหลัง

ลาป่วยย้อนหลังได้ 3 วัน · เกินนั้นต้องแนบใบรับรองแพทย์ + อนุมัติโดย HR ไม่ใช่ผู้จัดการสาขา

i
ถอนคำขอหลังอนุมัติ

หลัง state = approved ถอนได้ภายใน 24 ชม. → คืน balance · transition ไป cancelled · เก็บ audit

i
วันหยุดนักขัตฤกษ์

ระบบตัดวันหยุดออกจาก days_used อัตโนมัติ · แสดงเป็น readonly ใต้ field "จำนวนวัน"

i18n & accessibility

เน้นภาษาไทยเป็นหลัก · ความแตกต่างกับสคริปต์ละตินมีผลต่อ type metric, line-height, และ truncation

Locale rules
  • html lang="th" · ค่าวันที่ใช้ วันอังคาร · 21 เมษายน · ปี พ.ศ.
  • ตัวเลข KPI ใช้เลขฮินดู-อารบิก (1,840) เพื่อความเร็วในการอ่าน
  • Number formatting · Intl.NumberFormat('th-TH')
  • Currency: ฿ ที่หน้า · 0 ทศนิยม
A11y minimums
  • Body text contrast ≥ 4.5:1 (ink on cream = 14.6:1)
  • Tag/badge contrast ≥ 4.5:1 — soft variant ใช้ ink-2 จึงผ่าน
  • Focus ring 4px --accent-soft + 1px accent border
  • Hit target ≥ 36×36 (icon-btn = 40×40)
  • ใช้ semantic HTML — aside, button, kbd · ไม่มี div onClick ที่ไม่มี role

Dark theme ใช้ html[data-theme="dark"] override CSS variables · ไม่ duplicate component CSS · ใช้ color-mix(in oklab, …) เพื่อสร้าง soft tint อัตโนมัติ
